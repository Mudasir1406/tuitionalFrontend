## 1. Structured Data (JSON-LD Schemas)

### 1.1 Duplicate Organization Schema — FIXED

**Problem:** Two conflicting `Organization` schemas with the same `@id` (`https://tuitionaledu.com/#organization`) existed:
- `src/app/page.tsx` — had `foundingDate: "2022"`, location: Sharjah, rating: 4.4/100
- `src/app/ar/page.tsx` — had `foundingDate: "2022"`, location: Sharjah, rating: 4.7/108
- `src/components/seo/UniversalSchema.tsx` — had `foundingDate: "2020"`, location: Dubai, rating: 4.7/108

When Google sees two JSON-LD blocks with the same `@id` but conflicting data, it discards both.

**Fix:** Removed the inline `homeSchema` from `src/app/page.tsx` and `arabicHomeSchema` from `src/app/ar/page.tsx`. The single authoritative schema in `UniversalSchema.tsx` (loaded via root `layout.tsx`) now handles this for every page.

**Effect:** Google now has one clean, consistent Organization entity. No conflicts. Eligible for knowledge panel and rich results.

---

### 1.2 Invalid Schema Property Removed — FIXED

**Problem:** `UniversalSchema.tsx` had `"expertise": ["IGCSE", "A-Level", ...]` on the Organization. `expertise` is **not a valid Schema.org property** for Organization — Google ignores or penalises invalid properties.

**Fix:** Removed the `expertise` array entirely.

**Effect:** Schema validates cleanly against Schema.org spec. No validation warnings in Google Search Console.

---

### 1.3 Expired LocalBusiness Date — FIXED

**Problem:** `UniversalSchema.tsx` had `"validThrough": "2025-12-31"` — already past. An expired offer signals to crawlers the business may be inactive.

**Fix:** Updated to `"validThrough": "2027-12-31"`.

**Effect:** Offer schema stays valid for the next 2 years.

---

### 1.4 Unstable Logo URL — FIXED

**Problem:** The logo URL in `UniversalSchema.tsx` was `/_next/static/media/logo.16d39b17.png` — a hash-based path generated by Next.js at build time. This hash changes on **every rebuild**, breaking the canonical logo URL that Google stores.

**Fix:** Replaced all 7 occurrences with the stable public path: `/assets/images/static/logo.png`.

**Effect:** Google's indexed logo URL stays consistent across deployments. No more broken logo in rich results after each build.

---

### 1.5 Static Breadcrumb on Every Page — FIXED

**Problem:** `UniversalSchema.tsx` injected a single `BreadcrumbList` (`Home > About > Blog > Testimonials > Contact`) on **every single page** — including individual blog posts, course pages, etc. A blog post about IGCSE Maths should not show a breadcrumb trail of `Home > About > Blog > Testimonials > Contact`.

**Fix:** Removed the static BreadcrumbList from `UniversalSchema.tsx`. Created a new reusable `BreadcrumbSchema` component (`src/components/seo/BreadcrumbSchema.tsx`) that each page uses to inject its own correct breadcrumb.

**Effect:** Each page now has an accurate breadcrumb. Google can show page-specific breadcrumbs in search results.

### 1.6 BlogPosting Schema on Blog Posts — ADDED

**Files:** `src/app/blog/[slug]/page.tsx`, `src/app/ar/blog/[slug]/page.tsx`

**What was added:**
```json
{
  "@type": "BlogPosting",
  "headline": "...",
  "description": "...",
  "url": "...",
  "datePublished": "...",
  "dateModified": "...",
  "author": { "@type": "Organization", "name": "Tuitional" },
  "inLanguage": "en",
  "wordCount": 1234
}
```
- `datePublished` is computed from Firestore `timestamp.seconds`
- `wordCount` is computed by stripping HTML tags from `blogContent.content`
- Arabic version uses `inLanguage: "ar"`

**Effect:** Blog posts become eligible for Google's Article rich results (author byline, date, estimated reading info in SERPs).

### 2.2 Missing Hreflang on AR Pages — FIXED

Arabic pages had a `canonical` but no `languages` block, meaning Google didn't know the EN/AR relationship between pages.

**Pages fixed:**
- `src/app/ar/contact/page.tsx`
- `src/app/ar/testimonials/page.tsx`
- `src/app/ar/careers/page.tsx`
- `src/app/ar/online/page.tsx`
- `src/app/ar/pricing/page.tsx`
- `src/app/ar/about/page.tsx`

**Added to each:**
```ts
alternates: {
  canonical: `${SITE_URL}/ar/contact`,
  languages: {
    en: `${SITE_URL}/contact`,
    ar: `${SITE_URL}/ar/contact`,
    "x-default": `${SITE_URL}/contact`,
  },
},
```

**Effect:** Google correctly understands which EN and AR pages are equivalents. No duplicate content penalty between languages. Arabic search users get served the Arabic version.

## 5. Performance & Code Quality

### 5.1 Debug Console Logs Removed — FIXED

**`src/services/pricing/pricing-api.ts`** — ~50 `console.log` statements removed.

These were firing on every pricing page load and dumping:
- Raw Firestore `db` instance
- Project ID
- All document IDs and full document data
- Filter decision logs for every package

Any user with Chrome DevTools open could see internal Firebase configuration details. Also added server-side overhead logging data that was never used.

**`src/services/dropdown/dropdown-api.ts`** — 8 debug logs removed including Project ID exposure.

**`src/components/grade-subject-level/grade-subject-level.tsx`** — Removed 2 logs firing on every `/online/[slug]` page render (dumped entire page data object + every component name in the sequence).

**`src/components/grade-subject-level/main-content.tsx`** — Removed 1 log dumping content data on every render.

**`src/components/home/form-dialouge-v1.tsx`** — Removed `console.log("Invalid phone number!")` firing on every keystroke in the phone field.

**Effect:** 
- Internal Firebase config no longer visible to anyone with DevTools open
- Reduced server-side CPU time (logs are synchronous operations)
- Cleaner DevTools for developers — only real errors show now
- No risk of accidentally logging sensitive user data
